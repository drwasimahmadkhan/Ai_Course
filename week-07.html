Probability Fundamentals

Probability quantifies agent's belief about uncertain propositions. Bayes' rule enables updating beliefs from evidence.

Basic Concepts

Probability Space

Sample space Ω (all outcomes) + probability function P assigns value 0-1 to each outcome. P(Ω) = 1.

Key Rules

Joint Probability
P(A ∧ B) = probability both A and B true.
Conditional Probability
P(A|B) = P(A ∧ B) / P(B). Probability A given B known true.
Product Rule
P(A ∧ B) = P(A|B) P(B)
Bayes' Rule

P(A|B) = P(B|A) P(A) / P(B)
Core of probabilistic reasoning—relate posterior to likelihood × prior.

Bayesian Networks: Structure and Representation

Bayesian Networks compactly represent joint probability distributions using directed acyclic graphs (DAGs).

Key Insight

Factorization

Joint distribution factors as product of conditional probabilities:
P(X₁, ..., Xₙ) = Π P(Xᵢ | Parents(Xᵢ))

Network Components

⭕

Nodes

Random variables.

→

Directed Edges

Dependencies: A→B means A influences B.

📊

CPTs

Conditional Probability Tables for each node.

Compact Representation Network with n binary variables requires 2ⁿ joint entries without structure; Bayesian network uses far fewer parameters.

Exact Inference in Bayesian Networks

Computing posterior probabilities P(X|evidence) exactly.

Enumeration Algorithm

Process

For P(X|e):
1. Sum over hidden variables
2. For each combination, compute joint by multiplying CPT entries
3. Normalize result

Variable Elimination

Optimization

Cache intermediate results by eliminating variables one by one, factoring out sums intelligently. Exponential speedup vs. enumeration.

Complexity

  • Enumeration: O(2ⁿ) in worst case
  • Variable Elimination: O(n·k^(w+1)) where w = treewidth. Tractable for small w

Treewidth Measures network's tree-likeness. Low treewidth → efficient inference.

Approximate Inference: Sampling

For large networks, exact inference infeasible. Sampling approximates posteriors efficiently.

Prior Sampling

Algorithm

1. Sample each variable from its CPT given parent values
2. Repeat to generate samples
3. Estimate P(X|e) as fraction where X=x and evidence matches

Rejection Sampling

Improvement
Sample from prior, reject samples inconsistent with evidence. Keeps only valid samples.
Inefficiency
Rejects many samples if evidence unlikely. Exponential slowdown with evidence.

Likelihood Weighting

Better Approach

Fix evidence variables; sample rest. Weight each sample by likelihood of evidence. No rejection needed!

Convergence All sampling methods converge to true probabilities as samples → ∞ (Law of Large Numbers).

Markov Chain Monte Carlo (MCMC)

MCMC methods generate samples via Markov chain with stationary distribution = target posterior.

Gibbs Sampling

Algorithm

1. Initialize all variables randomly
2. Repeatedly: sample each variable from P(Xᵢ | current values of other variables)
3. After burn-in, samples approximate posterior

Advantages

  • Asymptotically correct: Samples converge to true posterior
  • Flexible: Works even when probabilities are unknown (unnormalized distributions)
  • Scalable: Can handle large networks

Challenges

  • Burn-in: Must discard initial samples before distribution stabilizes
  • Mixing time: How long until samples truly represent posterior?
  • Autocorrelation: Consecutive samples correlated; more samples needed

Temporal Models and Hidden Markov Models

Reasoning about sequences of events over time using Hidden Markov Models (HMMs).

HMM Components

Definition

• Hidden state sequence: X₁, X₂, ..., Xₜ
• Observation sequence: E₁, E₂, ..., Eₜ
• Transition model: P(Xₜ|Xₜ₋₁)
• Sensor model: P(Eₜ|Xₜ)
• Initial state: P(X₁)

Key Tasks

Filtering
Compute P(Xₜ|E₁:ₜ) from observations up to t. Real-time inference.
Smoothing
Compute P(Xₖ|E₁:ₜ) for k < t. Better estimates using future observations.
Most Likely Sequence
Find sequence X₁:ₜ maximizing P(X₁:ₜ|E₁:ₜ) given all observations.

Viterbi Algorithm Computes most likely sequence via dynamic programming.

Kalman Filters and Particle Filters

Specialized HMM algorithms for continuous state spaces.

Kalman Filter

For Linear Gaussian Models

Transition: Xₜ = FXₜ₋₁ + noise
Observation: Eₜ = HXₜ + noise
Update rules: recursively compute Gaussian posterior with optimal efficiency O(d³).

Applications

  • Tracking: Filter noisy sensor measurements to estimate object position/velocity
  • Navigation: Estimate robot pose from odometry + sensor fusion
  • Finance: Estimate stock trends from price observations

Particle Filters

For Nonlinear Models

Represent belief as set of weighted samples (particles). Resample based on likelihood. Generalizes to nonlinear, non-Gaussian systems.

Dynamic Bayesian Networks and Inference

Dynamic Bayesian Networks (DBNs) are Bayesian networks with temporal structure—foundation for temporal reasoning at scale.

DBN Structure

Representation

• Initial state: P(X₀)
• Transition: P(Xt | Xt-1)
• Sensor: P(Et | Xt)
• Repeat template for each time step

Inference Algorithms

Task Algorithm Complexity
Filtering Forward algorithm (sum-product) O(|X|²) per step
Smoothing Forward-backward (EM-like) O(|X|²) forward + O(|X|²) backward
Viterbi Dynamic programming max-product O(|X|²) per step

Key Advantage Fixed-size inference even as time sequences grow arbitrarily long!

Summary: Probabilistic Reasoning

Bayesian Networks

• DAG representation of dependencies
• Joint = product of CPTs
• Exact inference: elimination
• Approximate: sampling, MCMC

Temporal Models

• HMMs for discrete, DBNs for general
• Filtering, smoothing, most-likely-sequence
• Kalman filters for linear Gaussian
• Particle filters for nonlinear

Practical Guidance

  • Small networks: Exact inference (variable elimination)
  • Large networks, importance needed: Likelihood weighting or Gibbs sampling
  • Temporal sequences: Use forward algorithm (filtering) for real-time; Viterbi for explanations
