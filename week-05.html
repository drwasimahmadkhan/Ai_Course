Forward Chaining in FOL

Forward chaining applies inference rules to known facts, deriving new facts until reaching a goal or exhausting all inferences.

Algorithm Overview

Start with known facts. Apply rules: if rule premises match known facts, add conclusions. Repeat until goal found or no new facts derived.

Generalized Modus Ponens

Rule

For rule ∀x (P₁(x) ∧ P₂(x) ∧ ... ∧ Pₙ(x) → Q(x))
If P₁(a), P₂(a), ..., Pₙ(a) are true, derive Q(a)

Key Features

Completeness
For Horn clause databases, forward chaining is complete—derives all entailed facts.
Horn Clauses
Clauses with at most one positive literal: A ∨ ¬B ∨ ¬C. Much more tractable than general FOL.
Efficiency
Linear in KB size for many practical rule bases (Datalog programs).

Application Widely used in expert systems, deductive databases, and semantic web reasoning.

Backward Chaining and Proof Trees

Backward chaining works from goal backward to facts, building proof trees via goal decomposition.

Algorithm Steps

Backward Chaining Algorithm

1. Goal = query predicate
2. If goal is in KB, return proof
3. For each rule with goal as conclusion:
  • Unify goal with rule head
  • Recursively prove rule premises
4. Return proof tree if all premises succeed; backtrack if any fail

Comparison: Forward vs. Backward

Forward Chaining

• Data-driven approach
• Suitable when many facts, few queries
• Derives all possible inferences
• Good for monitoring, diagnosis

Backward Chaining

• Goal-driven approach
• Efficient for specific queries
• Only derives needed inferences
• Good for question answering

SLD Resolution Backward chaining uses SLD (Selective Linear Definite) resolution—foundation of Prolog!

Resolution in FOL: The Complete Framework

Resolution is a complete inference procedure for full FOL—can prove or disprove any sentence.

Resolution Refutation

Process

To prove α entailed by KB:
1. Convert KB ∧ ¬α to Conjunctive Normal Form (CNF)
2. Apply resolution until deriving empty clause
3. Empty clause = contradiction, so α must be true

CNF Conversion

  • Eliminate implications: α → β becomes ¬α ∨ β
  • Move negations inward: ¬(α ∧ β) becomes ¬α ∨ ¬β
  • Distribute OR over AND: (α ∧ β) ∨ γ becomes (α ∨ γ) ∧ (β ∨ γ)
  • Each resulting clause is disjunction of literals
✓

Complete

Can prove any entailed sentence in finite time.

⚡

Sound

Never derives false sentence.

🔄

Refutation

Proves by contradiction—powerful technique.

First-Order Logic Inference Challenges

FOL inference poses significant computational challenges compared to propositional logic.

Core Difficulties

Undecidability
No algorithm can determine truth/falsehood of all FOL sentences in finite time (Gödel, Church).
Semi-decidability
If sentence IS entailed, can prove it; if NOT entailed, may search forever.
Variable Instantiation
May need infinite substitutions to find proof (e.g., parent(X, Y) with infinite domain).
Clause Explosion
CNF conversion can exponentially increase clause count.

Restriction Horn clause restrictions, finite domains, and domain-specific heuristics make practical FOL reasoning feasible.

Knowledge Representation: Ontologies and Schemas

Structuring knowledge effectively is as important as inference. Ontologies define the concepts and relationships in a domain.

Ontology Components

📦

Entities/Objects

Domain objects: persons, places, things.

🏷️

Attributes

Properties of entities: color, height, age.

🔗

Relations

Connections between entities: parent-of, located-in.

🎯

Functions

Deterministic mappings: father-of(X), plus(X,Y).

Knowledge Base Organization

Approach Structure Advantage
Semantic Networks Nodes (concepts), edges (relations) Visual, intuitive, supports inheritance
Frames/Schemas Objects with slots, defaults, constraints Organized, efficient retrieval
Description Logics Formal ontology with subsumption Sound reasoning, automated classification
Rules/Clauses IF-THEN implications Flexible, compositional knowledge

Categories and Inheritance

Organizing entities into categories enables inheritance—properties of parent categories apply to children unless overridden.

Category Hierarchy

Example: Living Things

LivingThing ⊃ Animal ⊃ Mammal ⊃ Dog
Properties: LivingThing needs food → all animals need food

Inheritance and Exceptions

Default Inheritance
Assume typical properties apply unless explicitly stated otherwise. Birds fly—except penguins!
Multiple Inheritance
Entity can belong to multiple categories. Platypus: Mammal AND Egg-Layer.
Monotonic vs. Nonmonotonic
Classical logic: adding facts never removes conclusions. Nonmonotonic: exceptions can retract defaults.

Reasoning Challenge Handling exceptions, conflicts, and incomplete information requires default logic or circumscription.

Objects, Properties, and Relations

Fundamental building blocks for sophisticated knowledge representation in complex domains.

Composite Objects and Parts

Part-Whole Relationships

Objects composed of parts: Car has Engine, Wheels, Chassis. Supports reasoning about wholes through parts.

Events and Causality

Event Representation
Events occupy intervals: Flight(UA101, LAX, SFO) happens on date D. Supports temporal reasoning.
Causality
Event A causes event B: Crash(Flight) Causes Injury(Passenger).
Processes
Ongoing processes: Rain(Place, From, To). Processes have duration, agents, patients.

Measurement and Scale

  • Dimensions: Length, Weight, Temperature. Measured in units (meters, kilos, Celsius)
  • Conversion: 1 km = 1000 m. Scale relationships enable automated reasoning

Semantic Web and Description Logics

The Semantic Web applies formal knowledge representation to the internet, using Description Logics (DL) for machine-interpretable semantics.

Semantic Web Standards

🏷️

RDF

Resource Description Framework. Triple: subject-predicate-object.

📚

OWL

Web Ontology Language. Expressive DL for ontologies.

🔍

SPARQL

Query language for semantic data.

Description Logics

Concept

Formalize ontologies with automated reasoning: subsumption checking, classification, consistency checking.

Example Define: Doctor ⊑ Person, Surgeon ⊑ Doctor. Automatically infer: Surgeon ⊑ Person (subsumption).
