Learning and Inductive Learning

Learning improves performance through experience. Inductive learning infers general rules from specific examples.

Learning Component

Uses feedback (rewards/labels) to modify behavior, adjust parameters, refine models over time.

Supervised vs. Unsupervised

Supervised

Labeled examples: input X, output Y. Learn function f: X → Y. Classification, regression.

Unsupervised

Unlabeled data. Find structure: clusters, low-dim representations, distributions.

Key Concepts

  • Hypothesis: Candidate function mapping inputs to outputs
  • Generalization: Perform well on unseen test data, not just training
  • Bias-Variance: Trade-off between underfitting (high bias) and overfitting (high variance)

Decision Trees and Inductive Learning

Decision trees recursively partition feature space; interpretable, practical learning algorithm.

Algorithm: ID3 / C4.5

1. Choose best attribute (information gain)
2. Split dataset on attribute
3. Recursively build subtrees
4. Stop when pure or no good splits

Information Gain

Entropy
H(S) = -Σ p_i log p_i measures impurity of set S.
Information Gain
IG(S,A) = H(S) - Σ (|Sᵥ|/|S|)H(Sᵥ) — reduction in entropy from splitting on attribute A.

Strengths & Weaknesses

Strengths

• Interpretable
• Fast prediction
• Handles categorical/numeric
• Feature selection

Weaknesses

• Prone to overfitting
• Unstable (small changes → big trees)
• Biased toward balanced trees
• Limited expressiveness

Ensemble Methods and Boosting

Ensemble methods combine multiple learners for improved accuracy, robustness.

Ensemble Techniques

🌲

Bagging

Bootstrap samples; train separate models; average predictions. Reduces variance.

🔄

Boosting

Sequentially train models; focus on hard examples. Reduces bias; combines weak learners into strong.

🤝

Stacking

Train diverse base learners; meta-learner combines predictions. Flexible; captures interactions.

Boosting: AdaBoost

Algorithm

1. Initialize uniform weights
2. Train learner on weighted data
3. Increase weights of misclassified examples
4. Repeat, combine predictions via weighted majority

Result Exponential reduction in training error. Works if base learners better than random.

Evaluating Learned Hypotheses

Evaluation metrics assess model performance. Avoid overfitting via cross-validation, hold-out test sets.

Classification Metrics

MetricFormulaWhen to Use
Accuracy(TP+TN)/(TP+TN+FP+FN)Balanced classes
PrecisionTP/(TP+FP)Minimize false positives
RecallTP/(TP+FN)Minimize false negatives
F1 Score2·P·R/(P+R)Balanced precision-recall

Avoiding Overfitting

  • Cross-validation: K-fold trains K models on different splits. Averages performance
  • Regularization: Penalize model complexity; add L1/L2 terms to loss
  • Early stopping: Monitor validation loss; stop if not improving

Regression and Continuous Outputs

Regression predicts continuous values. Linear regression is simple baseline; polynomial/nonlinear for complex relationships.

Linear Regression

y = w₀ + w₁x₁ + ... + wₙxₙ
Minimize mean squared error: MSE = (1/m)Σ(y_predicted - y_actual)²
Closed-form solution via normal equations or gradient descent.

Nonlinear Extensions

Polynomial Regression
Include x², x³, ... as features. Captures nonlinear patterns; risk of overfitting.
Regularized Regression
Ridge (L2), Lasso (L1): penalize weights. Prevents overfitting; encourages sparsity (Lasso).

Evaluation Metrics

  • MSE: Mean squared error. Penalizes large errors
  • RMSE: Root MSE. Same units as output
  • R²: Coefficient of determination. Proportion of variance explained (0-1)

Unsupervised Learning: Clustering

Clustering groups similar examples without labels. Discovers natural structure in data.

K-Means Clustering

Algorithm

1. Initialize K cluster centers randomly
2. Assign each point to nearest center
3. Update centers as cluster means
4. Repeat until convergence

Clustering Algorithms

📊

K-Means

Fast; assumes spherical clusters. O(nkI) complexity.

🌳

Hierarchical

Dendrograms; flexible distances. O(n²) complexity.

🌐

DBSCAN

Density-based; handles arbitrary shapes. Finds core/border/noise points.

Evaluation (No Labels)

  • Silhouette Score: How well-separated/cohesive are clusters?
  • Davies-Bouldin Index: Cluster separation vs. scatter

Dimensionality Reduction

Dimensionality reduction projects high-dim data to lower dimensions, preserving structure.

Principal Component Analysis (PCA)

Approach

Find orthogonal directions (principal components) with maximum variance. Project onto top-K. Unsupervised; linear.

Other Techniques

t-SNE
Nonlinear; preserves local structure. Great for visualization; slow on large datasets.
Manifold Learning
Assumes data on low-dim manifold. Isomap, LLE discover intrinsic geometry.

Benefits

  • Visualization: 2D/3D projections for exploration
  • Noise reduction: Remove low-variance dimensions
  • Efficiency: Faster downstream learning

Summary: Learning from Examples

Supervised Learning

• Decision trees: interpretable
• Ensemble methods: robust
• Regression: continuous outputs
• Evaluation: train/test split, CV
• Trade-off: bias vs. variance

Unsupervised Learning

• Clustering: K-means, hierarchical
• Dimensionality reduction: PCA
• Manifold learning: nonlinear
• Evaluation: silhouette, intrinsic metrics
• Applications: exploration, preprocessing

Key Principles

  • Generalization: Good test performance, not just training
  • Overfitting: Regularization, cross-validation prevent
  • Ensemble: Combining weak learners → strong learner
