Knowledge and Logical Reasoning

Representing World Knowledge with Propositional and First-Order Logic

Artificial Intelligence: A Modern Approach (4th Ed.)

Logical Agents: The Knowledge-Based Approach

A knowledge-based agent maintains a knowledge base (KB) of facts and rules, using inference to derive new conclusions and guide actions.

Agent Architecture

• Sensors: Perceive environment
• Knowledge Base: Store facts, rules
• Inference Engine: Derive conclusions
• Actions: Execute based on inferences

Key Concepts

Percept
Observation from environment, added to knowledge base with timestamp.
Entailment
α ⊨ β means β is true in every model where α is true.
Inference
Process of deriving new sentences from known sentences.
Soundness
Only derive sentences that are entailed (no false conclusions).
Completeness
Can derive ANY sentence that is entailed (find all true conclusions).

Example If KB contains "All humans are mortal" and "Socrates is human", we can infer "Socrates is mortal".

Propositional Logic (PL)

Simplest logic: sentences are propositions that are either true or false. No objects or relationships.

Syntax and Semantics

Propositions
Atomic sentences: P, Q, R, etc. (true or false)
Operators
¬ (NOT), ∧ (AND), ∨ (OR), → (IMPLIES), ↔ (IFF)
Model
Assignment of true/false to each proposition
Truth Table
Shows truth value of compound sentence for all possible interpretations

Inference Techniques

📋

Modus Ponens

From α → β and α, derive β

⚡

Modus Tollens

From α → β and ¬β, derive ¬α

🔗

Unit Resolution

From α ∨ β and ¬β, derive α

Limitation Propositional logic cannot express relationships ("Person X likes Person Y") without exponential propositions.

First-Order Logic (FOL)

More expressive than PL: includes objects, predicates, quantifiers, and functions.

FOL Components

Constants
Objects: John, 5, MIT. Refer to specific entities.
Predicates
Relations: Parent(x,y), Brother(x,y), Prime(x). True/false for given objects.
Functions
Map objects: Father(John), Plus(2,3), Abs(x). Return objects, not truth values.
Variables
x, y, z. Bind to objects via quantifiers.
Quantifiers
∀x (for all), ∃x (there exists)

Examples of FOL Sentences

  • ∀x Parent(x, John) — "Everyone is parent of John" (obviously false!)
  • ∃x Parent(x, John) — "Someone is parent of John"
  • ∀x,y (Parent(x,y) → Ancestor(x,y)) — "If X parents Y, then X ancestors Y"
  • ∀x (Mortal(x) ↔ Human(x)) — "All and only humans are mortal"

Advantage FOL can express any logically computable fact about relationships and properties!

Inference in FOL: Unification and Resolution

Unification makes different syntactic forms represent same fact; Resolution is a powerful inference rule for FOL.

Unification Algorithm

Concept

Find substitution θ that makes two sentences identical. Example: Unify(Likes(x, Candy), Likes(John, y)) with θ = {x/John, y/Candy}.

Resolution Rule

General Form

From (α ∨ β) and (¬β ∨ γ), derive (α ∨ γ)
Applied with unification in FOL for refutation-based proof: assume goal is false, derive contradiction.

Backward Chaining vs. Forward Chaining

Backward Chaining

Goal-driven: start with goal, work backward to facts. Queries: "Does X follow from KB?"

Forward Chaining

Data-driven: start with facts, apply rules forward. Good for rule-based systems, diagnosis.

Completeness Resolution with unification is complete for FOL: any entailed sentence can be derived (though may take infinite time).

Knowledge Representation: Handling Complexity

Real-world knowledge is vast and complex. We need strategies to represent it efficiently and handle uncertainty.

Challenges

🌍

Ontology Commitment

What objects, properties, relationships does world contain?

⏰

Time and Change

How to represent time, causality, temporal relationships?

❓

Uncertainty

Handle incomplete, contradictory, or probabilistic information.

Representation Languages

  • Semantic Networks: Graph of concepts and relationships (nodes = concepts, edges = relations)
  • Frames: Objects with attributes and default values (house has rooms, doors, roof)
  • Rules: IF-THEN statements (if raining, then take umbrella)
  • Descriptions: Logical descriptions of object classes and properties

Key Idea Choose representation that makes inference efficient for typical queries.

Specific Knowledge: Facts and Reasoning Patterns

Categories and Taxonomies

Organize objects into categories with inheritance of properties. Example: Dog ⊂ Mammal ⊂ Animal

Inheritance

Properties of parent category apply to children UNLESS overridden. Mammals have fur (but not penguins)!

Composite Objects

Objects composed of parts with structure and relationships. Example: Car = Engine + Wheels + Chassis

Events and Time

Event Logic
Events occupy time intervals; can overlap, be concurrent, etc.
Causality
Event A causes event B. Important for planning and reasoning.
Temporal Reasoning
Before, after, during, meets relationships between events.

Summary: Knowledge and Logic

Propositional Logic

• Simple, decidable
• Limited expressiveness
• Modus Ponens, Resolution
• Truth tables for inference

First-Order Logic

• Rich expressiveness
• Objects, predicates, quantifiers
• Unification + Resolution
• Complete but undecidable

Inference Methods Comparison

Method Completeness Efficiency Use Case
Backward Chaining Complete Good for sparse queries Expert systems, diagnosis
Forward Chaining Complete Good when many facts Rule-based systems
Resolution Complete Systematic but slow Theorem proving
