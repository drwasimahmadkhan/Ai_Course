Classical Planning: STRIPS and Actions

Automated planning seeks sequences of actions achieving goals. STRIPS (Stanford Research Institute Problem Solver) is foundational.

Planning Problem

Initial state + Goal state + Set of actions → Find action sequence transforming initial → goal

Action Representation (STRIPS)

Preconditions
Logical conditions that must be true to execute action.
Effects
Positive (add facts) and negative (delete facts) effects on world state.
Example: Move(x, from, to)

Preconditions: At(x, from) ∧ Clear(to)
Effects: ¬At(x, from), At(x, to)

Closed World Assumption Anything not stated as true is false—simplifies reasoning.

Planning Algorithms: GraphPlan and SAT

Modern planners use planning graphs and SAT solvers for efficient solution finding.

GraphPlan Algorithm

Process

Build graph expanding facts/actions incrementally until goal facts all present with no mutex (mutual exclusion). Extract plan via backward search.

Planning via SAT

Approach

Encode planning problem as boolean satisfiability (SAT). For each time step t: encode preconditions, effects, mutex constraints as clauses. Solve with SAT solver (DPLL, CDCL).

Key Advantages

  • Complete: Find solution if one exists (within bounded steps)
  • Efficient: GraphPlan and SAT solvers handle larger problems than generic search
  • Flexible: Can handle various action types, temporal constraints

Comparison Planning graph typically faster than blind search; SAT planners competitive with specialized algorithms.

Uncertainty and Partial Observability

Real agents face incomplete information: unknown initial state, nondeterministic actions, partial observations.

Sources of Uncertainty

❓

Unknown State

Agent doesn't know exact initial conditions. Example: vacuum position unknown.

⚡

Nondeterminism

Actions have multiple possible outcomes. Moving may fail; sensing may be noisy.

👁️

Partial Observability

Can't see entire state; must infer from observations.

Belief States

Concept

Set of possible actual states consistent with agent's knowledge. Initially, belief state = all possible states.

Reasoning Agent maintains belief state; actions and observations narrow possibilities.

Sensorless and Contingency Planning

Agents must plan under uncertainty without relying on information they won't receive.

Sensorless (Conformant) Planning

Goal

Find action sequence succeeding in all possible initial states—guarantee goal regardless of hidden state.

Contingency Planning

Approach

Interleave planning and execution: observe outcomes, replan based on what's learned. Conditional plans: IF observation THEN action.

And/Or Graphs

OR Nodes
Choice of actions (agent control).
AND Nodes
Multiple outcomes (environment nondeterminism).

Execution Strategy Execute actions, observe results, follow plan branches matching observations.

Markov Decision Processes (MDPs)

MDPs model probabilistic environments where outcomes have known probabilities. Foundation for decision-making under uncertainty.

MDP Definition

Components

• States S: Possible configurations
• Actions A(s): Available actions per state
• Transition Model P(s'|s,a): Probability of reaching s' from s via a
• Rewards R(s,a,s'): Immediate reward for transition
• Discount factor γ: Weight future rewards (0 ≤ γ ≤ 1)

Value Functions

Function Meaning
V(s) Expected cumulative reward from state s following optimal policy
Q(s,a) Expected reward from taking action a in s, then following optimal policy
π(s) Optimal action to take in state s

Markov Property Future depends only on current state, not history—memoryless.

Value Iteration and Policy Iteration

Algorithms for computing optimal policies in MDPs.

Value Iteration

Algorithm

Iteratively improve value estimates:
V(s) = maxa Σs' P(s'|s,a) [R(s,a,s') + γ V(s')]
Repeat until convergence. Extract π(s) = argmaxa value.

Policy Iteration

Approach

1. Initialize policy π
2. Policy evaluation: compute Vπ (value under current π)
3. Policy improvement: π(s) = argmaxa Qπ(s,a)
4. Repeat until policy stable

Comparison

Value Iteration

• Combines evaluation & improvement
• Fewer iterations typically
• Good for online planning

Policy Iteration

• Explicit policy evaluation
• Faster convergence per iteration
• More stable for large state spaces

Partially Observable MDPs (POMDPs)

POMDPs extend MDPs to partial observability: agent can't see state directly, only receives observations.

POMDP Components

Observation Model
P(o|s) = probability of observing o in state s.
Belief Update
Bayes rule: update belief state b(s) = P(s|observations,actions).
Belief-Space MDP
Treat belief state as new "state" in MDP; plan in belief space.

Challenges

  • Belief space is continuous: Infinite possible belief distributions
  • Curse of history: Value function depends on entire observation history
  • Intractability: Optimal POMDP solving is PSPACE-hard

Approximation Use belief state compression, Monte Carlo sampling, or factored models for scalability.

Summary: Planning and Uncertainty

Classical Planning

• STRIPS actions with preconditions/effects
• GraphPlan: graph-based planning
• SAT planners: encode as satisfiability
• Complete solutions; deterministic world

Decision Under Uncertainty

• MDPs: probabilistic environments
• Value/Policy iteration: solve MDPs
• POMDPs: partial observability
• Belief-space planning

Key Decisions

  • Known environment: Use classical planning (GraphPlan, SAT)
  • Stochastic, fully observable: Use MDP solving (value/policy iteration)
  • Stochastic, partially observable: Use POMDP methods or contingency planning
