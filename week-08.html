Making Decisions

Decision Theory and Utility

Artificial Intelligence: A Modern Approach (4th Ed.)

Rational Preferences and Utility

Decision theory combines probability (beliefs) with utility (preferences) to determine optimal actions.

Expected Utility Principle

Choose action maximizing expected utility: E[U(a)] = Σ P(outcome|a) U(outcome)

Preferences and Axioms

Orderability
Agent can compare any two outcomes: A ≻ B, A ∼ B, or B ≻ A.
Transitivity
A ≻ B and B ≻ C implies A ≻ C.
Continuity
If A ≻ B ≻ C, ∃p: pA + (1-p)C ∼ B.

Foundation These axioms imply existence of utility function U mapping outcomes to real numbers.

Utility Functions and Preferences

Utility functions quantify agent's preferences over outcomes. Different agents have different utilities.

Money and Diminishing Returns

Non-Linear Utility

U(money) typically concave: each additional dollar worth less. Explains risk aversion and insurance purchasing.

Utility Assessment

Reference Lottery
Adjust probability p of best outcome until indifferent to certain mid-outcome: U(mid) = p·U(best) + (1-p)·U(worst).
Risk Aversion
Concave utility: prefer certain payment over gamble with same expected value.

Multi-Attribute Utility

Complex Preferences

Decompose into attributes: U(S) = f(U₁(S), U₂(S), ..., Uₙ(S))
Additive/multiplicative functions model interactions.

The Value of Information

Value of Perfect Information (VPI) quantifies worth of gaining certainty about uncertain variables.

Computing VPI

Formula

VPI(X) = Expected value WITH perfect info on X - Expected value WITHOUT
Shows value of reducing specific uncertainty.

Optimal Information Gathering

  • Gather information while VPI > cost
  • Continue experiments/sensing as long as expected value grows
  • Stop when marginal value < cost
🔍

Imperfect Info

Real sensors provide noisy observations. VPI generalizes to value of imperfect information.

🔗

Sequential Decisions

With partial observability, decide dynamically: sense, act, observe, decide again.

Decision Networks (Influence Diagrams)

Decision networks extend Bayesian networks with decision and utility nodes, graphically representing decision problems.

Components

🔷

Decision Node

Agent chooses action; represents decision point.

⭕

Chance Node

Random variable; represents uncertainty.

💎

Utility Node

Value function; shows payoff structure.

Evaluation Method

Algorithm

1. For each decision, compute expected utility given evidence
2. Choose action maximizing expected utility
3. Use backward induction for sequential decisions

Representation Graphically clear; computationally tractable for moderate-sized problems.

Markov Decision Processes (MDPs) and Sequential Decisions

MDPs model sequential decision problems with stochastic outcomes, solved using dynamic programming.

Components Revisited

  • States S: Configurations agent can be in
  • Actions A: Choices available per state
  • Transition model P(s'|s,a): Outcome probabilities
  • Rewards R(s): Immediate payoff per state
  • Discount γ: Weight future (0 ≤ γ ≤ 1)

Policy and Value Functions

Policy π(s)
Specifies action to take in each state.
Value V^π(s)
Expected cumulative reward under policy π starting from s.
Optimal π*
Maximizes V*(s) for all s.

Bellman Equation V*(s) = max_a Σ P(s'|s,a)[R(s) + γV*(s')]

Game Theory and Multiagent Decisions

Game theory extends decision theory to multiagent settings where outcomes depend on others' actions.

Normal Form Games

Matrix Representation

Players choose actions simultaneously; payoffs depend on action pairs. Example: Prisoner's Dilemma (Cooperate vs. Defect).

Nash Equilibrium

Definition
No player can improve unilaterally by changing strategy. Stable outcome.
Pure Strategy
Deterministic action choice.
Mixed Strategy
Randomize over actions; equilibrium exists for finite games (Nash's theorem).

Common Games

GameConflictExample
Zero-SumPerfect: one wins, other losesChess, poker
Prisoner's DilemmaIndividual incentive conflicts collective goodArms races, tragedy of commons
CoordinationMutual benefit from agreementStandards, conventions

Cooperative Game Theory

Coalition formation when agents cooperate to achieve common goals or split value.

Characteristic Function

Concept

v(S) = value achievable by coalition S independently. Defines game; agents form coalitions sharing total value.

Solution Concepts

Core
Allocations where no coalition wants to split off (stable).
Shapley Value
Fair allocation considering marginal contributions. Unique solution satisfying efficiency, symmetry, null-player axioms.
Nucleolus
Allocations minimizing maximum coalition dissatisfaction.

Application Allocating project costs, distributing profits, coalition formation in multiagent systems.

Preference Elicitation and Aggregation

Preference elicitation obtains user utilities; aggregation combines preferences for group decisions.

Elicitation Methods

📋

Questionnaire

Ask users about preferences directly. Simple but may not capture nuances.

🎰

Lottery Method

Gambles reveal risk attitudes. More expensive but detailed.

🛒

Ranking/Rating

Rank items or rate scales. Efficient; assumes transitivity.

Social Choice Theory

  • Voting: Plurality, approval, Condorcet. No perfect method (Arrow's impossibility)
  • Preference aggregation: Combine individual utilities (weighted sum, max-min fairness)
  • Mechanism design: Create incentives for truthful revelation of preferences

Summary: Decision-Making Under Uncertainty

Single Agent

• Utility functions encode preferences
• Expected utility principle
• Decision networks for representation
• Value of information guides sensing

Multiple Agents

• Game theory: strategic interaction
• Nash equilibrium: stable outcomes
• Coalition formation
• Mechanism design for incentives

Key Takeaways

  • Rationality: Maximize expected utility given beliefs and preferences
  • Value of information: Gather data strategically based on impact
  • Sequential decisions: Use MDPs; solve with dynamic programming
  • Multiple agents: Consider game-theoretic equilibria and coalition stability
