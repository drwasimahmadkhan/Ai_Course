Problem Formulation

The foundation of all search algorithms begins with precisely defining the problem.

Five Essential Components

A search problem consists of: initial state, actions, transition model, goal test, and action costs.

Key Components

Initial State
The starting condition of the problem, where the agent begins its search.
Actions
Set of possible actions available to the agent in any given state.
Transition Model
Describes how the world changes when an action is taken: Result(s, a).
Goal Test
Determines whether a state is a goal state (terminal/solution state).
Action Cost
The cost c(s, a, s') of taking action a from state s to state s'.

Real-World Examples

🗺️

Route Finding

Find shortest path between cities. States = cities, Actions = drive to adjacent city.

🧩

8-Puzzle

Arrange numbered tiles. States = tile positions, Actions = slide tiles.

🤖

Robot Assembly

Order parts assembly. States = partial assemblies, Actions = add/connect part.

State Space and Search Trees

State Space Graph

Definition

Nodes represent states; edges represent actions. The state space is the complete graph of all reachable states from the initial state.

Search Tree vs. State Space

State Space (Graph)

• Undirected or directed graph
• Shows all states and transitions
• Each state appears once
• Multiple paths may lead to same state

Search Tree

• Tree structure built during search
• Root = initial state
• Children = states reachable by actions
• Same state can appear multiple times in different branches

Key Terminology

  • Node: Data structure in search tree (has parent, children, state)
  • Frontier: Set of nodes ready to be expanded next
  • Reached set: Set of all nodes that have been encountered
  • Expanded node: Node whose children have been generated

Efficiency A search algorithm's efficiency depends on avoiding redundant paths and efficiently managing the frontier.

Uninformed (Blind) Search

Uninformed search algorithms use no knowledge about goal location, exploring the state space systematically.

Breadth-First Search (BFS)

Algorithm

Expand shallowest unexpanded node first. Uses a FIFO queue for the frontier.

Complete?
YES (if branching factor is finite)
Optimal?
YES (if all action costs are equal)
Time
O(bd) where b = branching factor, d = depth
Space
O(bd) — LARGEST PROBLEM!

Other Uninformed Strategies

Algorithm Complete Optimal Time Space
Depth-First No No O(bm) O(bm)
Iterative Deepening Yes Yes O(bd) O(bd)
Uniform Cost Yes Yes O(b⌈C*/ε⌉) O(b⌈C*/ε⌉)
Bidirectional Yes Yes O(bd/2) O(bd/2)

Key Insight Iterative deepening combines the space efficiency of depth-first with the completeness of breadth-first.

Informed (Heuristic) Search

Use domain knowledge to estimate distance to goal, dramatically improving search efficiency.

What is a Heuristic Function?

Definition

h(n) = estimated cost to reach goal from node n. A good heuristic guides search toward goal without guaranteeing optimality.

Properties of Good Heuristics

Admissibility
h(n) ≤ h*(n) for all n. The heuristic never overestimates true cost.
Consistency
h(n) ≤ c(n,a,n') + h(n'). Triangle inequality: estimate respects actual step costs.
Dominance
If h2(n) ≥ h1(n) for all n, then h2 dominates h1 (h2 is more informative).

Creating Heuristics

  • Relaxed problems: Remove constraints (e.g., Manhattan distance for 8-puzzle)
  • Pattern databases: Precompute costs for subproblems
  • Composite heuristics: Take maximum of multiple heuristics

Example For 8-puzzle: Manhattan distance = sum of distances each tile must move.

A* Search — The Gold Standard

A* combines actual cost and heuristic estimate for optimal, efficient search.

A* Evaluation Function

f(n) = g(n) + h(n)
• g(n) = actual cost from start to node n
• h(n) = estimated cost from n to goal
• f(n) = estimated total cost via n

Why A* is Optimal

Guarantees

A* is complete and optimal when using an admissible heuristic (on graph search) or consistent heuristic (on tree search).

A* Search Algorithm

Pseudocode

1. Start with frontier = {initial node}
2. While frontier not empty:
   • Pick node n with lowest f(n) = g(n) + h(n)
   • If n is goal, return path
   • Expand n, add children to frontier
3. Failure if frontier empty

Greedy Best-First Search

Uses only f(n) = h(n) (ignores g(n)). Faster but not optimal; often effective in practice.

Caution A* can use significant memory; use SMA* (Simplified Memory-bounded A*) for limited memory environments.

Search in Complex Environments (Ch. 4)

Local Search

When we only care about the final state, not the path, local search algorithms are very effective.

Hill Climbing
Always move to highest-value neighbor. Problem: can get stuck in local maxima.
Simulated Annealing
Probabilistically accept bad moves early, decrease probability over time. Escapes local maxima.
Genetic Algorithms
Population-based: mutation, crossover, selection. Good for complex fitness landscapes.
Min-Conflicts
For CSPs: randomly pick conflicted variable, assign value minimizing conflicts.

Online Search (Unknown Environments)

Agent Characteristics

Agent must act before learning entire state space. Discovers successor states and costs only when visiting them.

Key Algorithms

  • Online DFS: Memory O(d), but can take exponential time
  • Learning Real-Time A*: Updates heuristics as it learns; guarantees reaching goal

Application Robotics, game AI, and autonomous agents often use local/online search.

Search Complexity and Limitations

The Challenge

Combinatorial Explosion

Even moderate problems explode in size. Example: Chess branching factor ≈ 35, depth ≈ 100 → 35100 nodes!

Performance Measures

✓

Completeness

Algorithm guaranteed to find solution if one exists.

⚖️

Optimality

Algorithm finds least-cost solution.

⏱️

Time Complexity

Number of nodes expanded or evaluated.

💾

Space Complexity

Maximum number of nodes in memory.

Trade-offs

Priority Choose Reason
Small state space A* with good heuristic Optimal, complete, guided by knowledge
Memory limited Iterative Deepening or SMA* Linear space, still optimal
Only goal state matters Local Search Minimal memory, fast, no path needed
Anytime solution needed Greedy Best-First + A* Get answer fast, refine over time

Summary: Search Strategies

Uninformed vs. Informed

Uninformed search explores blindly; Informed search uses domain knowledge (heuristics) to guide exploration more efficiently.

Choose Your Strategy

Small Spaces
Use A* with Manhattan distance or similar heuristic for optimality and efficiency.
Memory Constrained
Use iterative deepening or SMA*; sacrifice time for space.
Goal State Only
Use hill climbing, simulated annealing, or genetic algorithms.
Unknown Environment
Use online search or learning real-time A*.

Key Concepts to Remember

  • Problem formulation is critical — accurate state space definition determines algorithm success
  • Heuristics dramatically improve search — must be admissible for A*
  • Trade-offs between completeness, optimality, time, and space are unavoidable
  • Domain knowledge is power — good heuristics > brute force search
